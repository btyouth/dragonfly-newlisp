Introduction

Dragonfly is a web framework for newLISP. Development started in June 2009. It focuses on speed, small memory consumption and a small learning curve. Other goodies are a plug and play architecture for writing own helpers or modules and a very easy deployment. It's possible to use it with the builtin newLISP webserver.

Ten reasons for tasting Dragonfly...

  1. You like newLISP
  2. You need ultra performance
  3. You want a framework with a small memory footprint
  4. You hate the rails -like deployment and just want a "transfer to webserver and go"
  5. You want a clear and easy-to-use templating language
  6. You want partial views in Your templates
  7. You want a framework which generates clean and search-friendly URLs
  8. You need a framework which is extensible
  9. You want a documented framework
  10. And discovering new programming styles
